Reticulate

About

Reticulate is an open source, secure, peer to peer file sharing system. Once a file has been uploaded to the Reticulate network only the files owner (and anyone else the file has been shared with) can access their content.

This means that no one can access your data without your express permission.

Getting Started

To get started download the latest release from the GitHub page.

  1. Extract the zip file
  2. Run either "bin/peer.bat" (Windows) or "bin/peer" (Mac and Linux)
  3. Enter a password for your Ethereum wallet (if you forget this there will be no way to recover your user account)
  4. Transfer Ether to the address listed under "User Address" (ether is required in order to complete write operations). You can get Ether by using the Rinkeby Faucet
  5. You will then be able to register a username.

How Does it Work?

Encryption

Each retiulate user has an accociated Ethereum wallet (on the Rinkeby test net) and an RSA key pair. When a file is uploaded a private key is generated that is used to encrypt the file, only someone with this key can access the file. The user that created the file encrypts this key with their RSA public key and stores that encrypted key on an Ethereum smart contract. Only that user can then decrypt the key later, in order to be able to access the file.

If Alice wants to share a file with Bob she can get her copy of the encrypted key from the contract, decrypt it and then re-encrpyt it with Bob's public key and add a second version to the contract.

Transfer

With a conventional file storage system files are stored in a single centralised location that is vulnerable to attack. On the Reticulate network files are stored by users of the network. Each Reticulate node stores some parts of some of the files on the network. When someone wants to download a file they download different parts of it from different nodes.
